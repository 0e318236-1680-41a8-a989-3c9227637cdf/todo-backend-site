Todo-Backend

a shared example to showcase backend tech stacks.

The Todo-Backend project helps showcase and compare different options for building web services. It defines a specification for a simple web API, and then provides example implementations for that same API using various tech stacks. The problem domain for this API is maintaining a list of todos. This problem domain - and the Todo-Backend project generally - was inspired by the TodoMVC project, and some code (specifically the web client implementation) was borrowed directly from TodoMVC.

the spec

The Todo-Backend API is specified by a suite of automated tests which run in a browser and can be pointed at any running Todo-Backend implementation. If your Todo-Backend implementation passes all the tests, then it is compliant with the spec.

a web client

In addition to the test specs there is also a web client which uses the API to maintain a list of todos. You can point the web client at any Todo-Backend implementation to play with it and manually verify that it performs as expected.
